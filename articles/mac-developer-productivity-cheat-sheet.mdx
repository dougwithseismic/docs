# Mac Developer Productivity Cheat Sheet

Stop clicking through menus. Stop typing the same commands. Here are the shortcuts, tools, and workflows that actually matter for Mac developers who want to ship faster.

<Info>
**Quick wins**: Set up these 5 shortcuts and you'll save 2+ hours per day: `Cmd+Space` for Spotlight, `Cmd+Shift+.` to show hidden files, `Cmd+K` in Finder for Go to Folder, terminal aliases for git, and Rectangle for window management.
</Info>

### Git Shortcuts You Need

Add these to your `~/.zshrc` or `~/.bashrc`:

```bash
# Git shortcuts that actually save time
gac() {
    if [ -z "$1" ]; then
        echo "Usage: gac 'commit message'"
        return 1
    fi
    git add . && git commit -m "$1"
}

gacp() {
    if [ -z "$1" ]; then
        echo "Usage: gacp 'commit message'"
        return 1
    fi
    git add . && git commit -m "$1" && git push
}

# Quick git status
alias gs='git status'
alias gd='git diff'
alias gl='git log --oneline -10'
alias gb='git branch'
alias gco='git checkout'
alias gcb='git checkout -b'
alias gp='git pull'
alias gps='git push'
```

**Usage:**
- `gac "fix typo"` - Add all changes and commit
- `gacp "add new feature"` - Add, commit, and push in one command

### Terminal Improvements

#### Oh My Zsh - Transform Your Terminal Experience

[Oh My Zsh](https://github.com/ohmyzsh/ohmyzsh) is a community-driven framework for managing your Zsh configuration. Think of it as a supercharged terminal that makes command-line work actually enjoyable.

**What it gives you:**
- **Smart autocompletion** - Tab completion for git commands, file paths, and more
- **Git integration** - Shows branch status, dirty state, and shortcuts in your prompt
- **Plugin ecosystem** - 300+ plugins for everything from Docker to AWS
- **Themes** - Beautiful, informative prompts that show what you need to know
- **Aliases** - Hundreds of useful shortcuts (like `gst` for `git status`)

**Install Oh My Zsh:**
```bash
sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
```

**Why it matters**: Instead of typing `git status` 50 times a day, you type `gst`. Instead of guessing git branch names, they autocomplete. Instead of a boring `$` prompt, you see exactly what branch you're on and if you have uncommitted changes.

<img src="/images/articles/cheatsheet/ohmyzsh.png" alt="Oh My Zsh with autosuggestions and autocomplete in action" />

**Essential Oh My Zsh Plugins:**

Install these powerful plugins to supercharge your terminal:

```bash
# Auto-suggestions (gray text completions from history)
git clone https://github.com/zsh-users/zsh-autosuggestions ~/.oh-my-zsh/custom/plugins/zsh-autosuggestions

# Syntax highlighting (colors commands red/green for invalid/valid)
git clone https://github.com/zsh-users/zsh-syntax-highlighting ~/.oh-my-zsh/custom/plugins/zsh-syntax-highlighting

# Fast syntax highlighting (improved performance version)
git clone https://github.com/zdharma-continuum/fast-syntax-highlighting ~/.oh-my-zsh/custom/plugins/fast-syntax-highlighting

# Real-time autocomplete (dropdown menus as you type)
git clone --depth 1 https://github.com/marlonrichert/zsh-autocomplete ~/.oh-my-zsh/custom/plugins/zsh-autocomplete
```

**Add to `~/.zshrc`:**
```bash
# Better command history
HISTSIZE=10000
SAVEHIST=10000
setopt HIST_IGNORE_DUPS
setopt HIST_FIND_NO_DUPS

plugins=(
  git
  zsh-autosuggestions
  zsh-syntax-highlighting
  fast-syntax-highlighting
  zsh-autocomplete
)
```

**What each plugin does:**
- **zsh-autosuggestions**: Press `→` to accept gray suggestions from your command history
- **zsh-syntax-highlighting**: Commands turn green when valid, red when invalid
- **fast-syntax-highlighting**: Faster, more accurate syntax coloring
- **zsh-autocomplete**: Real-time dropdown menus with tab completion

<Info>
**Note**: These plugins add ~1-2 seconds to terminal startup time, but the productivity gains from autosuggestions and syntax highlighting are well worth the brief wait. You'll save far more time than you lose.

You can mix and match any combination of these plugins—simply remove unwanted ones from your `plugins=()` list in `.zshrc` and run `source ~/.zshrc` to apply changes instantly.
</Info>

### Useful Functions

Add to `~/.zshrc`:

```bash
# Create directory and cd into it
mkcd() {
    mkdir -p "$1" && cd "$1"
}

# Extract any archive
extract() {
    if [ -f $1 ]; then
        case $1 in
            *.tar.bz2)   tar xjf $1     ;;
            *.tar.gz)    tar xzf $1     ;;
            *.bz2)       bunzip2 $1     ;;
            *.rar)       unrar e $1     ;;
            *.gz)        gunzip $1      ;;
            *.tar)       tar xf $1      ;;
            *.tbz2)      tar xjf $1     ;;
            *.tgz)       tar xzf $1     ;;
            *.zip)       unzip $1       ;;
            *.Z)         uncompress $1  ;;
            *.7z)        7z x $1        ;;
            *)           echo "'$1' cannot be extracted via extract()" ;;
        esac
    else
        echo "'$1' is not a valid file"
    fi
}

# Find and kill processes
killport() {
    lsof -ti:$1 | xargs kill -9
}

# Weather
weather() {
    curl "wttr.in/$1"
}
```

## Project Setup Shortcuts

### Quick Project Templates

```bash
# React project
create-react() {
    npx create-react-app $1 && cd $1 && code .
}

# Next.js project
create-next() {
    npx create-next-app@latest $1 && cd $1 && code .
}

# Node.js project
create-node() {
    mkdir $1 && cd $1
    npm init -y
    touch index.js .gitignore README.md
    echo "node_modules/\n.env\n*.log" > .gitignore
    code .
}

# Python project
create-python() {
    mkdir $1 && cd $1
    python -m venv venv
    source venv/bin/activate
    touch main.py requirements.txt .gitignore README.md
    echo "__pycache__/\n*.pyc\nvenv/\n.env" > .gitignore
    code .
}
```

## Final Setup

### Complete .zshrc Template

Here's a production-ready `.zshrc` setup:

```bash

plugins=(git zsh-autosuggestions zsh-syntax-highlighting)

# History

mkcd() { mkdir -p "$1" && cd "$1" }
```
