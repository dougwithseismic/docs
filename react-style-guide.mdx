# React Component Style Guide

This guide outlines the standards for creating interactive React components within our Mintlify documentation. All components should follow these patterns to ensure consistency, accessibility, and proper dark mode support.

## Core Design Principles

### 1. Container Structure

All interactive components should use a consistent container pattern:

```jsx
<div className="bg-background rounded-lg p-6 my-6 border border-neutral-500/20 shadow-sm">
  {/* Component content */}
</div>
```

For simpler components or those requiring less emphasis:

```jsx
<div className="p-4 border dark:border-zinc-950/80 rounded-xl not-prose">
  {/* Component content */}
</div>
```

### 2. Dark Mode Support

Always implement both light and dark mode styles using Tailwind's `dark:` modifier:

```jsx
// Text colors
className="text-gray-900 dark:text-gray-100"  // Primary text
className="text-gray-600 dark:text-gray-400"  // Secondary text
className="text-gray-500 dark:text-gray-500"  // Muted text

// Background colors
className="bg-white dark:bg-neutral-800/20"    // Input backgrounds
className="bg-neutral-500/5"                   // Subtle backgrounds
className="bg-gray-50 dark:bg-gray-900"        // Section backgrounds

// Border colors
className="border-gray-200 dark:border-neutral-500/20"
className="border-gray-200 dark:border-gray-700"
```

### 3. Typography Standards

#### Headers
```jsx
<h3 className="text-lg my-0 font-semibold text-gray-900 dark:text-gray-100">
  Component Title
</h3>
```

#### Labels
```jsx
<label className="block text-sm text-zinc-950/70 dark:text-white/70">
  Label Text
</label>
```

#### Small Text
```jsx
<span className="text-xs text-gray-600 dark:text-gray-400">
  Supporting text
</span>
```

### 4. Interactive Elements

#### Buttons

Primary action buttons:
```jsx
<button
  className="px-4 py-2 text-sm font-medium rounded-sm bg-orange-500 text-white hover:bg-orange-600 transition-all"
>
  Primary Action
</button>
```

Secondary/toggle buttons:
```jsx
<button
  className={`px-4 py-2 text-sm font-medium rounded-sm border transition-all ${
    isActive
      ? "bg-orange-500 text-white border-orange-500"
      : "bg-transparent text-gray-700 dark:text-gray-300 border-gray-200 dark:border-gray-700 hover:border-orange-400"
  }`}
>
  Toggle Button
</button>
```

#### Input Fields

Text inputs:
```jsx
<input
  type="text"
  className="w-full px-4 py-3 text-sm border border-gray-200 dark:border-neutral-500/20 rounded-sm outline-none focus:border-orange-500 dark:focus:border-orange-400 transition-colors bg-white dark:bg-neutral-800/20 text-gray-900 dark:text-gray-100"
  placeholder="Enter text..."
/>
```

Range sliders:
```jsx
<input
  type="range"
  className="w-full h-2 bg-zinc-950/20 rounded-lg appearance-none cursor-pointer dark:bg-white/20 mt-1"
/>
```

### 5. Spacing Standards

- Container padding: `p-6` for main containers, `p-4` for compact ones
- Section margins: `my-6` for component containers
- Element spacing: `mb-4` for major sections, `mb-2` for labels, `gap-3` for grids
- Inline spacing: `gap-2` for small elements, `gap-4` for larger ones

### 6. Card & Grid Patterns

Info cards:
```jsx
<div className="bg-neutral-500/5 p-3 rounded-sm border border-neutral-500/10">
  <div className="text-xs font-medium text-gray-700 dark:text-gray-300">
    Card Title
  </div>
  <div className="text-xs text-gray-600 dark:text-gray-400 mt-1">
    Card content
  </div>
</div>
```

Grid layouts:
```jsx
<div className="grid grid-cols-2 md:grid-cols-4 gap-3">
  {/* Grid items */}
</div>
```

### 7. Visual Feedback

#### Hover States
```jsx
// Scale on hover
className="transition-transform hover:scale-105"

// Border color change
className="hover:border-orange-400"

// Background change
className="hover:bg-orange-50 dark:hover:bg-orange-900/10"
```

#### Copy Confirmation
```jsx
const [copied, setCopied] = useState(false);

const handleCopy = async (text) => {
  try {
    await navigator.clipboard.writeText(text);
    setCopied(true);
    setTimeout(() => setCopied(false), 2000);
  } catch (err) {
    console.error("Failed to copy:", err);
  }
};

// Button shows confirmation
<button className={copied ? "bg-green-500 text-white" : "bg-orange-500 text-white"}>
  {copied ? "Copied!" : "Copy"}
</button>
```

### 8. Tooltips

Standard tooltip pattern:
```jsx
<div className="group relative">
  <div className="cursor-help">Hover me</div>
  <div className="absolute bottom-full left-1/2 transform -translate-x-1/2 mb-2 px-3 py-2 bg-gray-900 dark:bg-gray-800 text-white text-xs rounded-sm opacity-0 group-hover:opacity-100 transition-opacity pointer-events-none z-10 min-w-[150px] shadow-lg">
    <div className="font-semibold mb-1">Tooltip Title</div>
    <div className="text-gray-300">Tooltip description</div>
    <div className="text-gray-400 text-[10px] mt-1">Additional info</div>
  </div>
</div>
```

### 9. Special Elements

#### Progress Bars
```jsx
<div className="w-full bg-gray-200 dark:bg-gray-700 rounded-full h-2">
  <div
    className="h-2 bg-gradient-to-r from-purple-500 to-indigo-500 rounded-full transition-all duration-500"
    style={{ width: `${percentage}%` }}
  />
</div>
```

#### Alert/Info Boxes
```jsx
<div className="mb-4 p-4 bg-purple-50 dark:bg-purple-900/20 rounded-sm border border-purple-200 dark:border-purple-800">
  <div className="flex items-start gap-2">
    <span className="text-xl">⚠️</span>
    <div>
      <div className="text-sm font-semibold text-purple-700 dark:text-purple-400 mb-1">
        Alert Title
      </div>
      <div className="text-xs text-purple-600 dark:text-purple-400">
        Alert message content
      </div>
    </div>
  </div>
</div>
```

### 10. Animation & Transitions

- Use `transition-all` for general transitions
- Use `transition-colors` for color-only changes
- Use `transition-transform` for scale/rotate changes
- Standard duration: default (150ms) or `duration-500` for slower animations
- Loading states: `animate-pulse` for skeleton screens

### 11. Responsive Design

Always consider mobile-first design:
```jsx
// Grid columns
className="grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4"

// Text size
className="text-sm md:text-base"

// Padding
className="p-3 md:p-6"
```

### 12. Accessibility

- Always include proper labels for form elements
- Use semantic HTML elements
- Include `title` attributes for interactive elements
- Ensure sufficient color contrast
- Add `disabled` states with `cursor-not-allowed`

## Component Checklist

Before finalizing any React component for documentation:

- [ ] Dark mode support implemented
- [ ] Responsive on mobile devices
- [ ] Hover states for interactive elements
- [ ] Copy-to-clipboard functionality where relevant
- [ ] Consistent spacing using standard classes
- [ ] Proper TypeScript types (if applicable)
- [ ] Error handling for async operations
- [ ] Loading states for data fetching
- [ ] Tooltips for complex interactions
- [ ] Consistent border radius (`rounded-sm` or `rounded-lg`)

## Example Component Structure

```jsx
import React, { useState, useEffect } from 'react';

export const ExampleComponent = () => {
  const [isLoading, setIsLoading] = useState(false);
  const [data, setData] = useState(null);

  // Loading state
  if (isLoading) {
    return (
      <div className="bg-gray-50 dark:bg-gray-900 rounded-lg p-6 my-6 animate-pulse">
        <div className="h-4 bg-gray-200 dark:bg-gray-700 rounded w-1/3"></div>
      </div>
    );
  }

  return (
    <div className="bg-background rounded-lg p-6 my-6 border border-neutral-500/20 shadow-sm">
      {/* Header */}
      <div className="flex items-end justify-between mb-4">
        <h3 className="text-lg my-0 font-semibold text-gray-900 dark:text-gray-100">
          Component Title
        </h3>
      </div>

      {/* Content */}
      <div className="space-y-4">
        {/* Your component content */}
      </div>

      {/* Footer */}
      <div className="mt-6 pt-6 border-t border-gray-200 dark:border-gray-700">
        <div className="text-sm text-gray-600 dark:text-gray-400">
          Footer content
        </div>
      </div>
    </div>
  );
};
```

## Color Palette Reference

### Brand Colors
- Primary: `orange-500` (#F97316)
- Primary Hover: `orange-600` (#EA580C)
- Primary Light: `orange-50`, `orange-900/10` (dark mode)

### Status Colors
- Success: `green-500`, `green-600`
- Warning: `yellow-500`, `yellow-600`
- Error: `red-500`, `red-600`
- Info: `purple-500`, `indigo-500`

### Neutral Colors
- Backgrounds: `neutral-500/5`, `neutral-500/10`, `neutral-500/20`
- Borders: `neutral-500/10`, `neutral-500/20`
- Text: Follow the gray scale from `gray-900` to `gray-400`

## Mintlify Integration

Remember that these components exist within Mintlify's documentation system:

1. Use `not-prose` class to disable Mintlify's default typography when needed
2. Components should be self-contained and not rely on external state
3. Avoid conflicts with Mintlify's built-in styles
4. Test in both light and dark themes
5. Ensure components work with Mintlify's navigation and page transitions

## Performance Considerations

- Use `useMemo` for expensive calculations
- Implement proper cleanup in `useEffect` hooks
- Limit re-renders with proper state management
- Keep bundle size minimal - avoid large external dependencies
- Use lazy loading for heavy components

## Testing Your Components

Before deploying:

1. Test in both light and dark modes
2. Check responsive behavior at different breakpoints
3. Verify keyboard navigation works properly
4. Test copy-to-clipboard functionality
5. Ensure all animations are smooth
6. Check for console errors
7. Validate accessibility with screen readers
